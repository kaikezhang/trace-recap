## 一、产品定位

### 一句话
一个让普通旅行者在 3 分钟内把旅行路线变成精美动画短视频的 App。

### 目标用户
- 普通旅行者（非专业剪辑师）
- 想在朋友圈 / Instagram / 抖音 / 小红书分享旅行 recap
- 追求简单快速 + 好看，不需要复杂编辑
- 核心诉求：**快、美、分享**

### 核心体验
用户完成一次旅行后，打开 App → 添加走过的城市 → 选交通方式 → 贴几张照片 → 一键导出短视频。全程 < 3 分钟。

### 竞品参考
- **TravelBoast** — 可爱卡通图标风格（核心竞品）
- **Mult.dev** — 干净的 2D 地图动画美感
- **Polarsteps** — 旅行轨迹记录（但太重，不是短视频工具）

## 三、功能模块详解

### 3.1 地图

**底图：**
- 2D 平面地图（非 3D 地球）
- 3 种地图样式可切换：
  - **浅色** — 浅灰白底，适合大多数场景
  - **深色** — 深灰黑底，夜间感/高级感
  - **卫星** — 卫星影像 + 道路标注
- 支持手势操作：双指缩放、单指拖拽、双指旋转（可选）

**地图上显示的元素：**
- 目的地标记点（marker）
- 路线连线（不同交通方式用不同颜色和线型）
- 播放时显示的动画元素（交通工具图标、城市名标签）

**路线线型样式：**

| 交通方式 | 线条颜色 | 线型 |
|---------|---------|------|
| 飞机 | 靛蓝色 | 虚线 |
| 汽车 | 琥珀色 | 实线 |
| 火车 | 翠绿色 | 实线 |
| 巴士 | 紫色 | 实线 |
| 轮渡 | 青蓝色 | 短虚线 |
| 步行 | 粉色 | 点线 |
| 自行车 | 蓝绿色 | 长虚线 |

### 3.2 添加目的地

**方式一：搜索添加**
- 搜索框输入城市/地点名称
- 调用地理编码 API（Mapbox / Apple MapKit）返回搜索建议列表
- 用户点击建议项 → 添加到路线末尾
- 搜索支持中英文

**方式二：地图点击添加**
- 在地图空白处长按（或点击）→ 弹出确认，添加该坐标为目的地
- 自动反向地理编码获取地点名称

**方式三：AI 自然语言生成（可选增强功能）**
- 用户输入一段文字描述旅行，例如 "我从北京坐飞机去东京，然后坐新干线去京都，再坐火车去大阪"
- AI 解析生成完整路线（目的地 + 交通方式）
- 用户可在生成后手动调整

**目的地数据结构：**
- 名称（英文）
- 名称（中文，自动获取）
- 经纬度坐标
- 是否为途经点（waypoint）
- 照片列表（0-9 张）
- 照片排版设置

### 3.3 路线列表（目的地排序）

**功能：**
- 以列表形式显示所有目的地，按路线顺序排列
- 每个目的地之间显示交通方式选择器和时间控件
- 目的地支持拖拽重新排序
- 删除某个目的地后，前后目的地自动重新连线

**每个目的地卡片显示：**
- 序号
- 地点名称
- 照片缩略图（如有）
- 途经点标识（如已标记）
- 照片数量提示

**目的地之间的连接区域显示：**
- 交通方式选择器
- 段落时间（自动计算值 或 用户 override 值）

### 3.4 交通方式选择

**7 种交通方式：**
1. ✈️ 飞机（Flight）
2. 🚗 汽车（Car）
3. 🚆 火车（Train）
4. 🚌 巴士（Bus）
5. ⛴️ 轮渡（Ferry）
6. 🚶 步行（Walk）
7. 🚲 自行车（Bicycle）

**路线几何生成规则：**
- 飞机 → 大圆弧线（great circle arc），弧形飞行路线
- 汽车 / 巴士 → 真实公路路线（Directions API，驾车模式）
- 火车 → 近似公路路线（Directions API，驾车模式，无铁路专用数据）
- 轮渡 → 贝塞尔曲线（水面弧线）
- 步行 → 步行路线（Directions API，步行模式）
- 自行车 → 骑行路线（Directions API，骑行模式）

**交互：**
- 点击交通方式图标 → 弹出选择器，展示 7 个选项
- 切换后自动重新生成该段路线几何

### 3.5 途经点（Waypoint）

**概念：**
途经点是路途中经过但不停留的地方。动画播放时，摄像机会飞越途经点但不会暂停、不显示城市名、不显示照片。

**功能：**
- 任何目的地可以被标记 / 取消标记为途经点
- 标记为途经点后：
  - 该点前后的两段路线合并为一个动画组（merged group）
  - 摄像机连续飞行，不在该点停留
  - 不显示城市名标签
  - 不显示照片
- 取消途经点后恢复为正常停靠站

**用途举例：**
北京 → **郑州**（途经）→ 武汉：摄像机从北京一路飞到武汉，经过郑州但不停，飞行路线沿铁路走。

### 3.6 照片管理

**照片添加：**
- 每个目的地最多 9 张照片
- 从相册选择 / 拍照添加
- 支持批量选择
- 照片存储为本地文件引用（或项目内复制）

**照片操作：**
- 删除单张照片
- 调整照片焦点（focal point）：用户拖拽确定照片裁剪中心点（0-1 范围的 x, y 坐标），影响 cover 裁剪时保留的区域
- 添加 caption（可选文字说明）

**照片在动画中的表现：**
- 当动画到达某个有照片的目的地时，地图暂停
- 照片以卡片形式出现在画面上
- 显示约 2 秒后，照片以动画方式收起
- 然后继续下一段动画

### 3.7 照片排版

**两种模式：**

#### 自动排版（Auto）
系统根据照片数量和方向自动选择最佳布局：

| 数量 | 布局规则 |
|------|---------|
| 1 张 | 单张居中，最大化显示 |
| 2 张 | 并排显示 |
| 3 张 | 1 大 + 2 小（主图占左 2/3，右侧两张各占一半高度） |
| 4 张 | 2×2 网格 |
| 5 张 | 1 大 + 4 小（主图占上 60%，下面 4 张平分） |
| 6 张 | 2×3 或 3×2（根据照片方向决定） |
| 7 张 | 1 大 + 6 小 |
| 8 张 | 2 大 + 6 小 |
| 9 张 | 3×3 九宫格 |

#### 手动排版（Manual）
用户可选择 5 种排版模板：

1. **Grid（网格）** — 均匀网格排列
2. **Hero（主图）** — 一张大图 + 其余小图
3. **Masonry（瀑布流）** — 高低错落的瀑布流风格
4. **Filmstrip（胶片）** — 横向电影胶片条
5. **Scatter（散落）** — 随机角度散落，像桌上的实体照片

**手动排版的可调参数：**
- 照片顺序：拖拽调整
- 间距（Gap）：0-20px
- 圆角（Border Radius）：0-20px
- 自定义比例（高级）：行高比例、列宽比例

**排版编辑交互：**
- 用户点击某个目的地的照片区域 → 进入排版编辑状态
- 地图跳转到该目的地，显示照片预览效果
- 用户调整排版参数，所见即所得
- 确认后保存

### 3.8 段落时间控制

**自动计算（默认）：**
- 系统根据目的地数量自动计算总时长和每段时长
- 总时长范围：20 秒 ~ 180 秒
- 每段时长按路线距离比例分配
- 短行程更紧凑，长行程节奏稍慢

**手动 Override：**
- 用户可以为任意一段路线设置自定义时长
- 调整范围：2 秒 ~ 30 秒
- 设置后该段使用固定时长，其余段重新按比例分配
- 可随时恢复为自动计算
- 一键清除所有 override

**每段动画包含 5 个阶段，比例固定：**

| 阶段 | 说明 | 时间分配 |
|------|------|---------|
| Hover | 停留在出发城市 | 固定 ~1.5s |
| Zoom Out | 摄像机拉远 | 可变时间 × 20% |
| Fly | 交通工具沿路线移动 | 可变时间 × 65% |
| Zoom In | 摄像机推近目的地 | 可变时间 × 15% |
| Arrive | 停留在目的地 | 固定 ~1.5s + 照片时间 |

### 3.9 城市名标签

**显示时机：**
- Hover 阶段：显示出发城市名
- Arrive 阶段：显示到达城市名
- 途经点不显示

**可调参数：**
- 语言：英文 / 中文（切换显示）
- 字号：12px ~ 48px 可调
- 标签出现时有动画效果（pop / fade in）

### 3.10 播放预览

**功能：**
- 播放 / 暂停按钮
- 重置（回到起点）
- 进度条：可拖拽跳转到任意时间点
- 显示当前时间 / 总时长

**播放时的画面效果：**
- 摄像机按 5 阶段动画移动
- 交通工具卡通图标沿路线移动（4 个方向变体：上下左右，根据行进方向切换）
- 路线在 Fly 阶段逐渐绘制（animated route drawing）
- 城市名标签在 Hover/Arrive 阶段弹出
- 照片在 Arrive 阶段显示，在下一段开始时以优雅动画收起（缩小 + 上浮 + 模糊）

**交通工具图标：**
- 每种交通方式有 4 个方向的卡通图标（right, down, left, up）
- 根据当前行进方向实时切换：
  - 北 (315°-45°) → up
  - 东 (45°-135°) → right  
  - 南 (135°-225°) → down
  - 西 (225°-315°) → left
- 图标跟随路线移动，大小固定

### 3.11 视频导出

**导出参数：**
- 画面比例：16:9（横屏）或 9:16（竖屏 / Stories）
- 分辨率：720p 或 1080p
- 帧率：30fps
- 城市标签语言（同预览设置）
- 城市标签字号（同预览设置）

**导出流程：**
1. 用户点击导出 → 选择参数
2. 系统逐帧捕获动画画面（地图 + 交通图标 + 城市标签 + 照片，全部合成到一张画面上）
3. 所有帧编码为 H.264 MP4
4. 导出完成后用户可下载 / 保存到相册

**导出进度：**
- 显示进度条：当前帧数 / 总帧数
- 分阶段提示：捕获帧 → 编码视频 → 完成
- 支持取消

**导出后：**
- 保存到相册
- 直接分享到 Instagram Stories / 微信朋友圈 / 其他 App（调用系统分享）

### 3.12 路线导入 / 导出

**导出路线（JSON）：**
- 将当前路线保存为 JSON 文件
- 包含所有目的地、交通方式、段落时间 override、照片排版设置
- 不包含照片文件本身（照片仅含 URL 引用或本地路径）

**导入路线（JSON）：**
- 从文件导入之前导出的路线
- 导入后自动生成路线几何（调用 Directions API）
- 自动获取中文地名

**JSON 格式示例：**
```json
{
  "name": "台湾环岛",
  "locations": [
    {
      "name": "Taipei",
      "nameZh": "台北",
      "coordinates": [121.5654, 25.0330],
      "isWaypoint": false,
      "photos": [
        { "url": "...", "caption": "台北101" }
      ],
      "photoLayout": {
        "mode": "auto"
      }
    }
  ],
  "segments": [
    { "fromIndex": 0, "toIndex": 1, "transportMode": "train" }
  ],
  "timingOverrides": {
    "seg_1": 10
  }
}
```

## 四、动画引擎详细规格

### 4.1 摄像机行为

**Hover 阶段：**
- 摄像机静止在出发城市上方
- 缩放级别：城市级（zoom 10-16，根据前一段距离自动计算）
- 持续约 1.5 秒

**Zoom Out 阶段：**
- 摄像机从城市级缩放拉远到航线级
- 缩放级别：根据两城市距离自动计算（距离越远 zoom 越小）
- 使用 ease-out 缓动曲线（快速开始，平滑减速）

**Fly 阶段：**
- 摄像机跟随交通工具图标移动
- 摄像机中心沿路线几何插值
- 缩放级别保持在航线级
- 线性匀速移动（无缓动）

**Zoom In 阶段：**
- 摄像机从航线级推近到目的地城市级
- 使用 ease-in-out 缓动曲线（平滑进出）

**Arrive 阶段：**
- 摄像机静止在目的地上方
- 若有照片：地图保持不动，照片层覆盖显示
- 持续约 1.5 秒 + 照片显示时间（~2 秒/张）

### 4.2 缩放自动计算

```
飞行缩放级别 = clamp(14 - log2(距离km / 50), 2, 10)
城市缩放级别 = clamp(飞行缩放 + 4, 10, 16)
```

- 短距离（< 50km）：飞行 zoom ≈ 14，城市 zoom ≈ 16
- 中距离（~500km）：飞行 zoom ≈ 10.7，城市 zoom ≈ 14.7
- 长距离（> 5000km）：飞行 zoom ≈ 7.3，城市 zoom ≈ 11.3
- 洲际（> 10000km）：飞行 zoom ≈ 2-5，城市 zoom ≈ 10

### 4.3 动画组合并（Waypoint Merging）

当两个目的地之间存在途经点时，多段路线合并为一个动画组：
- A → **B**（途经）→ C：合并为一个 A→C 的动画组
- 摄像机从 A 的 Hover 直接飞到 C 的 Arrive，中间不停
- 飞行路线沿 A→B→C 的实际路径走
- 缩放级别根据 A 和 C 的距离计算（而非 A→B 或 B→C）

### 4.4 路线绘制动画

- 在 Fly 阶段，路线线条不是一开始就全部显示
- 而是随着交通工具图标的移动逐渐"画出来"
- 从出发点开始，图标走到哪里线就画到哪里
- 到达时路线完整显示

### 4.5 照片退出动画

当照片需要收起时（下一段动画开始）：
- 每张照片按顺序依次收起（staggered exit），不是同时消失
- 收起效果：缩小 + 向上飘移 + 高斯模糊 + 微旋转
- 在下一段的 Hover → Zoom Out 阶段完成退出
- 整体照片层有透明度渐隐

```
│   └── geometry: LineString? (路线几何，运行时生成)

```

- BGM / 背景音乐

- 3D 地球视图

### 🔮 后续可扩展
- BGM 背景音乐选择

## 八、技术约束（供设计师了解）

- **地图渲染**：MapKit 或 Mapbox SDK (iOS)，WebGL / Metal 渲染，支持自定义样式和相机控制
- **动画帧率**：预览和导出均为 30fps
- **视频编码**：设备本地编码（AVAssetWriter / Metal），无需服务器
- **照片存储**：本地沙盒，不上传云端
- **离线能力**：地图需要网络加载瓦片，路线几何需要网络查询 Directions API，其余功能离线可用

## 九、关键交互场景描述

### 场景 1：首次使用
用户刚下载 App，没有任何项目。
→ 引导用户创建第一个项目 → 搜索第一个城市 → 搜索第二个城市 → 选交通方式 → 点播放预览 → 觉得不错就导出

### 场景 2：一次完整编辑
1. 创建新项目 "台湾环岛"
2. 搜索添加：台北、台中、台南、嘉义、阿里山、台北
3. 台中→台南选"火车"，嘉义→阿里山选"巴士"，其余默认"火车"
4. 给台北添加 3 张照片（101、夜市、淡水）
5. 给阿里山添加 2 张照片（日出、小火车）
6. 点预览，觉得阿里山那段太快 → 手动调到 8 秒
7. 切到中文标签，字号调大到 24px
8. 选 9:16 竖屏，1080p 导出
9. 保存到相册 → 分享到朋友圈

### 场景 3：导入已有路线
1. 朋友分享了一个 trace-recap JSON 文件
2. 用户通过 "导入" 功能打开文件
3. 路线自动加载，所有目的地和交通方式就位
4. 用户替换成自己的照片
5. 预览 → 导出

### 场景 4：复杂路线（含途经点）
用户规划了一个 10 城市的欧洲行，其中 3 个城市只是转车经过。
→ 把 3 个转车城市标记为途经点 → 动画流畅飞过不停留 → 只在 7 个真正停留的城市展示照片和城市名
