# 照片排版设计方案

## 需求
- 每个地点最多 **9 张照片**（当前限制 5 张）
- 支持两种排版模式：**智能自动排版** + **手动排版**
- 排版结果同时影响预览和导出

---

## 一、智能自动排版（Auto Layout）

### 设计理念
参考小红书/Instagram 的九宫格 + Google Photos 的智能拼贴。根据照片数量和方向（横/竖）自动选择最佳布局。

### 排版规则

| 数量 | 布局 | 说明 |
|------|------|------|
| 1 | 单张居中 | 最大化显示，横图 80% 宽，竖图 50% 宽 |
| 2 | 并排 / 上下 | 同方向并排，混合方向则横上竖下 |
| 3 | 1 大 + 2 小 | 主图占左 2/3，右侧两张各占 1/2 高 |
| 4 | 2×2 网格 | 均分四格 |
| 5 | 1 大 + 4 小 | 主图占上 60%，下面 4 张平分 |
| 6 | 2×3 / 3×2 | 根据整体方向决定行列 |
| 7 | 1 大 + 6 小 | 主图占上，下面 2 行×3 列 |
| 8 | 2 大 + 6 小 | 上面 2 张大，下面 2 行×3 列 |
| 9 | 3×3 网格 | 经典九宫格 |

### 自动排版算法
1. 分析每张照片的 aspect ratio
2. 分类：横图 (>1.1)、竖图 (<0.9)、方图
3. 按预定义模板匹配最佳布局
4. 每张照片在分配的格子内 `object-fit: cover`（裁剪填满）

## 二、手动排版（Manual Layout）

### 交互设计（参考 Instagram Layout + PicCollage）

#### 核心交互
1. **拖拽排序** — 长按照片拖动到新位置，其他照片自动让位（类似 iOS Home Screen）
2. **模板选择** — 提供 3-5 个预设模板供用户选择（网格、拼贴、杂志风等）
3. **调整分割线** — 拖动照片之间的分割线调整比例（类似 Instagram Layout app）

#### 模板库（按照片数量）

**通用模板类型：**
- `grid` — 均匀网格（默认）
- `hero` — 一张大图 + 其余小图
- `masonry` — 瀑布流式高低错落
- `filmstrip` — 横向电影胶片条
- `scatter` — 随机散落风格（类似实体照片）

**每种数量提供 3 个模板选择，用缩略图 icon 展示。**

#### 手动模式 UI

```
┌─────────────────────────────┐
│ [Auto ✨] [Grid] [Hero] [Scatter] │  ← 模板选择条
├─────────────────────────────┤
│                             │
│   [照片排版预览区]            │  ← 可拖拽排序
│   拖拽照片调整位置            │
│                             │
├─────────────────────────────┤
│ Gap: [━━━●━━] 0-20px       │  ← 间距滑块
│ Radius: [━━━●━━] 0-20px    │  ← 圆角滑块
└─────────────────────────────┘
```

#### 详细交互流程

1. **默认**: Auto 模式（智能排版，一键搞定）
2. **点击其他模板**: 切换到手动模式，应用选中模板
3. **拖拽排序**: 在预览区长按照片 → 拖到新位置 → 松手应用
4. **调整间距**: 拖动 Gap slider 调整照片间间距（0-20px）
5. **调整圆角**: 拖动 Radius slider 调整照片圆角（0-20px）

## 三、数据模型

```typescript
// 新增到 Location 类型
interface Location {
  // ...existing fields
  photos: Photo[];
  photoLayout?: PhotoLayout; // 如果 undefined，使用 Auto
}

interface PhotoLayout {
  mode: "auto" | "manual";
  template?: LayoutTemplate; // manual 模式下选中的模板
  order?: string[];          // 照片 ID 排序（拖拽后的顺序）
  gap?: number;              // 间距 px (0-20)
  borderRadius?: number;     // 圆角 px (0-20)
}

type LayoutTemplate = 
  | "grid"      // 均匀网格
  | "hero"      // 一大多小
  | "masonry"   // 瀑布流
  | "filmstrip" // 横向胶片
  | "scatter";  // 随机散落
```

## 四、排版编辑交互

### 入口
点击 LocationCard（任意位置）→ 两件事同时发生：
1. **时间轴跳转**到该地点的 ARRIVE 阶段（地图上显示该地点 + 照片 overlay）
2. **地图区域上浮出排版编辑面板**

### 排版编辑面板（浮在地图区域上方）

```
┌──────────┬──────────────────────────────┐
│ Left     │           Map Area           │
│ Panel    │                              │
│          │  ┌──────────────────────────┐│
│ [Seoul]  │  │  📷 Seoul — Photo Layout ││
│ [ATL] ←──│──│                          ││
│ [...]    │  │  [Auto✨] [Grid] [Hero]  ││
│          │  │  [Masonry] [Scatter]     ││
│          │  │                          ││
│          │  │  ┌──┬──┬──┐             ││
│          │  │  │  │  │  │  ← 排版预览  ││
│          │  │  ├──┼──┼──┤  可拖拽排序  ││
│          │  │  │  │  │  │             ││
│          │  │  └──┴──┴──┘             ││
│          │  │                          ││
│          │  │  Gap: [━━━●━━] 8px      ││
│          │  │  Radius: [━━●━━] 8px    ││
│          │  │                 [Done ✓] ││
│          │  └──────────────────────────┘│
└──────────┴──────────────────────────────┘
```

### 交互流程
1. 用户点击左侧 LocationCard
2. 动画引擎 seekTo 该地点的 ARRIVE 时间点 → 地图飞到该城市
3. 排版编辑面板从右侧/底部滑入，覆盖在地图上方（半透明背景）
4. 面板内上方是模板选择条（icon 按钮），下方是排版预览（可拖拽排序）
5. 预览区同步更新地图上的 PhotoOverlay（所见即所得）
6. 调整完毕点击 Done → 面板收起，排版保存到 Location.photoLayout
7. 导出时 VideoExporter 读取 photoLayout 渲染

### 时间轴跳转逻辑
- 点击 LocationCard index=i → 找到 segments[i-1] 的 ARRIVE 阶段起始时间
- 第一个地点（index=0）→ 跳到 timeline[0] 的 HOVER 阶段
- 调用 engine.seekTo(arriveTime / totalDuration)
- 设置 playbackState = "paused"

## 五、实现优先级

### Phase 1（MVP）
- [ ] 照片上限从 5 改到 9
- [ ] 实现 Auto 智能排版（上面的排版规则表）
- [ ] 预览 (PhotoOverlay) 和导出 (VideoExporter) 同步使用新排版

### Phase 2
- [ ] 手动模板选择 UI（模板选择条 + 预览）
- [ ] 拖拽排序
- [ ] Gap / Border Radius slider

### Phase 3（可选）
- [ ] Scatter 模式（随机角度、位置）
- [ ] 自定义分割线拖拽调整比例
- [ ] 照片裁剪/缩放（在格子内调整 focal point）
