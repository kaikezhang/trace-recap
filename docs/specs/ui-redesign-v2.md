# TraceRecap UI Redesign v2 — Design Spec

## 目标

从"功能堆叠的工具"升级为"简洁美观、交互流畅的产品"。

核心设计原则：
- **减法设计**：默认只显示核心操作，高级功能渐进展示
- **呼吸感**：足够的留白、柔和的过渡、不拥挤
- **一致性**：统一的圆角、间距、动画时长
- **专注模式**：编辑时看编辑面板，播放时全屏看地图

---

## 1. Landing Page 重设计

### 现状问题
- 太简陋：一个标题 + 两个按钮
- 没有产品说明、没有预览、没有信任感

### 设计方案

```
┌────────────────────────────────────────────────────┐
│                                                    │
│  ✈️ TraceRecap                                     │
│                                                    │
│  Turn your travel routes into                      │
│  cinematic animated videos.                        │
│                                                    │
│  [▶ Try Demo]   [+ New Journey]                    │
│                                                    │
│  ┌──────────────────────────────────┐              │
│  │                                  │              │
│  │    Looping GIF / 短视频预览       │              │
│  │    (自动播放一段台湾旅行动画)      │              │
│  │                                  │              │
│  └──────────────────────────────────┘              │
│                                                    │
│  3 步说明卡片:                                      │
│  ① 添加城市    ② 自定义路线    ③ 导出视频          │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 具体要求
- 背景：subtle 渐变或地图纹理，不要纯白
- Demo 按钮用 primary 色（主 CTA），New Journey 用 outline
- 3 步说明卡片用 icon + 一句话，不要多
- Hero 区域放一个自动播放的动画预览（可以是录好的 GIF 或嵌入一个 mini 地图动画）
- 移动端：卡片竖排，预览视频占满宽度

## 2. Editor 整体布局重构

### 现状问题
- TopToolbar 按钮太多太挤（undo/redo/import/export/save/clear/style/toggle）
- LeftPanel 功能重复（Import/Save 同时出现在 TopToolbar 和 LeftPanel）
- 没有视觉层级：所有按钮同等重要
- 搜索框藏在 panel 里不够显眼

### Desktop 新布局

```
┌──────────────────────────────────────────────────────┐
│ ✈️ TraceRecap          [Undo] [Redo]    [⋮ More]    │  ← 极简顶栏
├────────────────┬─────────────────────────────────────┤
│                │                                     │
│  🔍 搜索城市   │         MAP (全屏地图)               │
│                │                                     │
│  ─────────────│                                     │
│  城市列表      │                                     │
│  （可折叠卡片） │                                     │
│               │                                     │
│               │                                     │
│               │      [城市标签动画]                   │
│               │                                     │
│               │                                     │
│               │                                     │
│               │  ┌─────────────────────────────┐    │
│               │  │ ⏮  ▶  ⏭  ━━━●━━━━ 0:12/0:30│    │
│               │  └─────────────────────────────┘    │
├────────────────┴─────────────────────────────────────┤
```

### TopToolbar 精简

**保留在顶栏的：**
- Logo（可点击回首页）
- Undo / Redo（小图标，灰色禁用态）
- "⋮ More" 下拉菜单（包含：Import Route / Save Route / Clear Route / Export Video / Map Style）

**移出顶栏的：**
- Import / Save Route → 移到 More 菜单
- Clear Route → 移到 More 菜单
- Map Style 选择器 → 移到 More 菜单
- Export Video → 移到 More 菜单 **或** 作为固定底部 CTA

### LeftPanel 精简

**移除的：**
- Import / Save Route 按钮（已在 More 菜单里）
- "Route" 标题头（多余）

**保留 & 优化的：**
- 搜索框 → 更大、带 placeholder 动画（"Search Tokyo, Paris, New York..."）
- 城市列表 → 卡片样式优化（见第 3 节）
- 宽度从 320px → 360px（给卡片更多空间）

## 3. 城市卡片（LocationCard）重设计

### 现状问题
- 信息太密集：英文名、中文名、照片缩略图、拖拽手柄、删除、waypoint 切换全挤在一起
- 照片管理入口不明显
- 编辑中文名的交互不直觉（要点击文字才能编辑）

### 新设计

```
┌────────────────────────────────────────┐
│ ⋮⋮  ① Taipei 台北                   ✕ │  ← 拖拽手柄 + 序号/名字 + 删除
│     ┌───┐┌───┐┌───┐  [+ 📷]           │  ← 照片缩略图（最多3个）+ 添加
│     │   ││   ││   │                    │
│     └───┘└───┘└───┘                    │
└────────────────────────────────────────┘
     │
     ✈️ 2,100 km                          ← 交通选择器（不变，已经很好）
     │
┌────────────────────────────────────────┐
│ ⋮⋮  ② Taichung 台中                 ✕ │
│     No photos  [+ Upload]              │
└────────────────────────────────────────┘
```

### 变化
- 英文名 + 中文名合并到一行（节省纵向空间）
- 照片缩略图行：显示前 3 张小圆角缩略图 + "+" 按钮
- 点击卡片 → 展开详细编辑（中文名编辑、照片排版、waypoint 切换）
- 默认收起 = 干净；展开 = 完整功能
- Waypoint 切换移到展开区域的 toggle

## 4. 搜索栏增强

### 现状问题
- 搜索框太小，placeholder 固定 "Search city..."
- 搜索结果样式简陋

### 新设计
- 搜索框高度增大到 44px（移动端友好的 touch target）
- placeholder 动画轮播："Search Tokyo...", "Search Paris...", "Search New York..."
- 搜索结果卡片带国旗 emoji 和国家名
- 空状态：搜索框下方显示 "Add your first stop to begin" 的 muted 提示

---

## 5. 播放控件升级

### 现状问题
- 桌面端：浮动在地图下方，小小的一条，不够精致
- 移动端：宽度撑满、按钮太小、进度条太细

### Desktop 新设计

```
┌─────────────────────────────────────────────────┐
│                                                 │
│  ⏮   ▶   ⏭      ━━━━━━━━●━━━━━━━━   0:12/0:30 │
│                                                 │
└─────────────────────────────────────────────────┘
```

- 圆角更大（rounded-2xl）
- 背景毛玻璃效果加强
- 按钮用圆形容器（play 按钮更大，44x44px）
- 进度条更粗（h-2 → h-3），hover 时变成 h-4
- 进度条上方显示当前段标签（如 "Taipei → Taichung"）

### Mobile 新设计
- 固定在底部，全宽
- Play 按钮居中、更大（56x56px 圆形、primary 色）
- 进度条在 play 按钮上方，占满宽度
- 时间显示在进度条两端

---

## 6. 移动端 BottomSheet 优化

### 现状问题
- 展开/收起不够流畅
- 高度固定 60vh 太死板
- 容易误触地图

### 新设计
- **3 个状态**：collapsed (只看搜索栏 + stops 计数) / half (搜索 + 路线列表) / full (搜索 + 路线 + 照片编辑)
- **手势拖拽**：用 framer-motion drag 而非简单 click toggle
- **collapsed 状态**：只显示搜索栏 + "3 stops" badge，高度约 120px
- **half 状态**：显示搜索 + 路线卡片列表，高度 50vh
- **full 状态**：同上 + 选中卡片的照片编辑，高度 85vh
- 背景遮罩只在 full 状态出现

---

## 7. 导出对话框优化

### 现状问题
- 对话框里设置项太多（aspect ratio, resolution, label language, label size）
- 导出进度条样式简陋

### 新设计
- 分两步：
  1. **预览 + 基本设置**：aspect ratio 用大按钮切换（横屏/竖屏两个卡片对比），resolution 默认 720p 不单独显示
  2. **高级设置**折叠区域：城市标签语言、大小、resolution
- 导出进度：用更优雅的圆环进度 + 动画
- 完成后：大大的 ✓ + 下载按钮，明确告诉用户文件大小
- "Quick Export (720p)" 按钮，一键导出不需要设置

## 8. 交互动画规范

### 统一过渡时长
| 类型 | 时长 | 缓动 |
|------|------|------|
| 展开/折叠 | 300ms | ease-out |
| 淡入淡出 | 200ms | ease |
| 弹出/弹入 | 350ms | spring(300, 25) |
| hover 反馈 | 150ms | ease |

### 统一圆角
| 组件 | 圆角 |
|------|------|
| 卡片 | 12px (rounded-xl) |
| 按钮 | 8px (rounded-lg) |
| 播放条 | 16px (rounded-2xl) |
| Bottom sheet | 20px top (rounded-t-2xl) |
| 输入框 | 8px (rounded-lg) |

### 统一间距
- 组件间：16px
- 卡片内 padding：12px (mobile) / 16px (desktop)
- section 间：24px

## 9. 颜色 & 主题

### 现状
- 只有 light 主题（代码里没有 dark mode 支持）
- 基本用 shadcn 默认色

### 方案
- 保持 light 模式为默认
- 品牌色：indigo-500 作为 primary（路线画线颜色已经是 indigo 系）
- 卡片背景：比 background 稍暗一点的 surface 色
- 增加微妙的渐变：TopToolbar 底部有一条细细的渐变分割线
- 地图区域背景：在地图加载前显示 skeleton loading 动画（而非空白）

## 11. 实现优先级

### Phase 1 — 布局 & 骨架 (PR29)
- TopToolbar 精简 + More 下拉菜单
- LeftPanel 精简（去掉重复按钮）
- 统一圆角、间距、动画时长

### Phase 2 — 卡片 & 搜索 (PR30)
- LocationCard 重设计（收起/展开模式）
- 搜索框增强（大输入框、placeholder 动画）
- 空状态设计

### Phase 3 — 播放 & 移动端 (PR31)
- PlaybackControls 升级（更大按钮、更精致样式）
- BottomSheet 三态优化（拖拽手势）
- 移动端整体适配

### Phase 4 — 导出 & Landing (PR32)
- ExportDialog 两步流程
- Landing Page 重设计（Hero + 3 步说明 + 预览）

## 12. 技术约束

- 使用现有 shadcn/ui 组件库，不引入新依赖
- framer-motion 已有，可用于动画
- Tailwind CSS class 为主，避免 inline style
- 移动断点：md (768px)
- 所有现有功能必须保留，只是重新组织 UI
- 不改动 store 逻辑和动画引擎
